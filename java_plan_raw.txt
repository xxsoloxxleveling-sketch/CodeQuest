DAY 01
What is Java & Your First Program
■ W H AT  Y O U  W I L L L E A R N
Java is a programming language used everywhere — phones, banks, big websites. Java code is first 
compiled (translated) and then run by the JVM (Java Virtual Machine), which is why Java runs on almost
any device.
Every Java program lives inside a class, and starts running from a special main  method. To show text
you use System.out.println() . Every statement ends with a semicolon ; .
■ C O D E  E X A M P L E
public class Main {
public static void main(String[] args) {
System.out.println("Hello, world!");
    }
}
The main method is where every Java program begins.

Variables & Primitive Types
■ W H AT  Y O U  W I L L L E A R N
A variable stores a value. In Java you must state the type when you create one — Java is strict about
this.
The main primitive types are: int  (whole numbers), double  (decimals), boolean  (true/false), and 
char  (a single letter). For words you use String .
Pattern: type name = value;
■ C O D E  E X A M P L E
int age = 12;
double price = 4.99;
boolean isReady = true;
String name = "Sara";
System.out.println(name);
Every variable in Java declares its type up front.

DAY 03
Numbers and Arithmetic
■ W H AT  Y O U  W I L L L E A R N
Java does math with + , - , * , / . A key rule: int / int  gives a whole-number result (it drops the
decimal).
So 7 / 2  is 3 , not 3.5. If you want the decimal, at least one number must be a double : 7.0 / 2
gives 3.5.
■ C O D E  E X A M P L E
int apples = 8;
int friends = 4;
int each = apples / friends;
System.out.println("Each gets " + each);
int division drops the decimal — a classic Java surprise.

DAY 04
Strings in Java
■ W H AT  Y O U  W I L L L E A R N
A String holds text and always uses double quotes "" . You join strings with +  — this is called 
concatenation.
When you add a number to a String with + , Java automatically turns the number into text: "Score: " +
50  becomes "Score: 50" .
■ C O D E  E X A M P L E
String first = "Code";
String last = "Quest";
String title = first + " " + last;
System.out.println(title);
+ glues strings (and numbers-as-text) together.

DAY 05
Comments & Reading Input (Scanner)
■ W H AT  Y O U  W I L L L E A R N
Comments are notes Java ignores: //  for one line, /* ... */  for several.
To read what a user types, Java uses a Scanner. You create one linked to System.in , then call methods
like nextLine()  for text or nextInt()  for whole numbers.
■ C O D E  E X A M P L E
import java.util.Scanner;
// ...inside main:
Scanner sc = new Scanner(System.in);
System.out.print("Your name? ");
String name = sc.nextLine();
System.out.println("Welcome, " + name);
Scanner is Java's tool for listening to the user.

DAY 06
Type Casting (Converting Types)
■ W H AT  Y O U  W I L L L E A R N
Casting means converting a value from one type to another. To turn a double  into an int , write the
target type in brackets: (int) 3.9  gives 3 (it cuts the decimal).
To turn text into a number, use helper methods: Integer.parseInt("42")  and 
Double.parseDouble("3.5") .
■ C O D E  E X A M P L E
double d = 9.7;
int i = (int) d;          // 9
int n = Integer.parseInt("100");
System.out.println(i + n);
Casting bridges types so they can work together.

■ W H AT  Y O U  W I L L L E A R N

DAY 08
More Operators (Modulo & Increment)
■ W H AT  Y O U  W I L L L E A R N
%  is the modulo operator — it gives the remainder of a division. 17 % 5  is 2. It is perfect for checking
even/odd: n % 2 == 0  means even.
++  adds 1 to a variable; --  subtracts 1. +=  and -=  add or subtract any amount in shorthand.
■ C O D E  E X A M P L E
int total = 17;
int boxes = total / 5;
int left = total % 5;
System.out.println(boxes + " boxes, " + left + " left");
/ and % team up to split things into groups.

DAY 09
Constants & final
■ W H AT  Y O U  W I L L L E A R N
Sometimes a value should never change — like the number of seconds in a minute. Mark it with the 
final  keyword to make it a constant.
If you try to change a final  variable, Java refuses to compile. By convention, constant names are
written in ALL_CAPS so they stand out.
■ C O D E  E X A M P L E
final int MAX_HP = 100;
final double PI = 3.14159;
System.out.println("Max HP is " + MAX_HP);
// MAX_HP = 200; -> compile error
final locks a value so it can never be changed by mistake.

DAY 12
Comparison Operators
■ W H AT  Y O U  W I L L L E A R N
Comparisons produce booleans. Java uses ==  equal, !=  not equal, > , < , >= , <= .
Important: for comparing Strings use .equals() , not == . ==  compares numbers and true/false;
.equals()  compares the actual text.
■ C O D E  E X A M P L E
int score = 75;
System.out.println(score > 50);   // true
String a = "hi";
System.out.println(a.equals("hi"));
Numbers use ==; Strings use .equals().

DAY 13
The if / else Statement
■ W H AT  Y O U  W I L L L E A R N
The if  statement runs a block only when its condition is true. The condition goes in round brackets,
and the block goes in curly braces { } .
Add an else  block for what happens when the condition is false. Java uses braces, not indentation, to
mark the block — but you should indent anyway for readability.
■ C O D E  E X A M P L E
int temp = 35;
if (temp > 30) {
System.out.println("It is hot!");
} else {
System.out.println("Comfortable.");
}
Braces mark exactly what belongs to the if and else.

DAY 14
else-if Chains & Nested if
■ W H AT  Y O U  W I L L L E A R N
To test several conditions in order, chain them with else if . Java checks each from the top and runs
the first true one.
You can also place an if inside another if  — this is a nested if, used when one decision depends on
another.
■ C O D E  E X A M P L E
int score = 82;
if (score >= 90) {
System.out.println("Grade A");
} else if (score >= 75) {
System.out.println("Grade B");
} else {
System.out.println("Keep going");
}
The first matching branch wins; the rest are skipped.

DAY 15
The switch Statement & Ternary
■ W H AT  Y O U  W I L L L E A R N
When you check one variable against many fixed values, a switch  is cleaner than a long if-chain. Each 
case  handles one value; break  ends it; default  catches the rest.
The ternary operator is a one-line if: condition ? valueIfTrue : valueIfFalse .
■ C O D E  E X A M P L E
int day = 3;
switch (day) {
case 1: System.out.println("Mon"); break;
case 3: System.out.println("Wed"); break;
default: System.out.println("Other");
}
switch picks one case quickly; remember the break.

Goal: Match an incoming rune to its effect with a switch.
Setup: A rune machine must react differently to each of several rune symbols.

DAY 16
The while Loop
■ W H AT  Y O U  W I L L L E A R N
A while  loop repeats its block as long as the condition stays true. The condition is checked before each
round.
You must change something inside the loop so the condition eventually becomes false — otherwise it
runs forever (an infinite loop).
■ C O D E  E X A M P L E
int count = 1;
while (count <= 5) {
System.out.println("Step " + count);
    count++;
}
count++ moves the loop toward its stopping point.

DAY 17
The for Loop
■ W H AT  Y O U  W I L L L E A R N
A for  loop is best when you know how many times to repeat. It packs three parts into one line: start, 
condition, and update.
for (int i = 0; i < 5; i++)  means: start i at 0, keep going while i is below 5, add 1 each round.
■ C O D E  E X A M P L E
for (int i = 1; i <= 5; i++) {
System.out.println("Counting: " + i);
}
Start, condition, update — all in one tidy line.
■ TO D AY ' S  G A M E  L E V E L
★ The Stepping Stones

Recursion
■ W H AT  Y O U  W I L L L E A R N
Recursion is a method that calls itself to solve a smaller version of the problem. It needs a base case —
a simple stopping condition — or it never ends.
Classic example: factorial. 5! = 5 * 4!  down to 1! = 1 .
■ C O D E  E X A M P L E

}
// factorial(5) -> 120
The method shrinks the problem until the base case stops it.

private locks the field; public methods are the safe door.

DAY 58
Polishing & Handling Mistakes
■ W H AT  Y O U  W I L L L E A R N
A working program is good; a polished one is great. Polishing means making it pleasant and hard to
break.
Add clear messages so the user always knows what is happening. Wrap risky parts in try-catch .
Check user input before trusting it. Use neat formatting in your output.
Try to break your own program on purpose — type wrong things — and fix whatever crashes. A program
that survives misuse is a finished program.
■ C O D E  E X A M P L E
try {
int choice = Integer.parseInt(input);
// ...act on the menu choice...
} catch (NumberFormatException e) {
System.out.println("Please type a number.");
}
Guard every input; a polished program rarely crashes.
