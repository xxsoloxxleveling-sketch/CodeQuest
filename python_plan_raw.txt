DAY 01
What is Programming & Your First Print
■ W H AT  Y O U  W I L L L E A R N
A computer is like a very fast helper that only does exactly what you tell it. Programming is writing those
instructions in a language the computer understands. Python is one of the easiest languages to learn — it
reads almost like English.
Your first instruction is print() . It tells Python: "show this message on the screen." Whatever you put
inside the quotation marks appears as text. Quotes tell Python "this is words, not a command."
Think of print()  as the computer speaking out loud. Every big program — games, apps, websites — is
built from tiny clear steps like this one.
■ C O D E  E X A M P L E
# This is your first Python program
print("Hello, world!")
print("I am learning to code.")
Each line is one instruction. Python runs them top to bottom.

DAY 02
Variables — Boxes That Hold Things
■ W H AT  Y O U  W I L L L E A R N
A variable is a labeled box where you store a piece of information so you can use it later. You create one
with the =  sign: the name goes on the left, the value goes on the right.
For example, name = "Sara"  puts the word Sara into a box labelled name . Later you just say name
and Python brings back what's inside.
You can change what's in the box any time — that is why it is called a variable (it can vary). Good box
names are short and describe what they hold, like score  or player_name .
■ C O D E  E X A M P L E
name = "Sara"
age = 12
print(name)
print("You are", age, "years old")
Notice: text needs quotes, numbers do not.

DAY 03
Numbers and Math
■ W H AT  Y O U  W I L L L E A R N
Python is a great calculator. It works with two kinds of numbers: integers (whole numbers like 7) and 
floats (numbers with a decimal point like 3.5).
The math symbols are simple: +  add, -  subtract, *  multiply, /  divide. You can store the result of a
calculation directly in a variable.
You can also do math using variables. If coins = 10 , then coins + 5  gives 15. The computer never
gets tired of counting!
■ C O D E  E X A M P L E
apples = 8
friends = 4
each = apples / friends
print("Each friend gets", each, "apples")
Division with / always gives a float (4.0, not 4).

DAY 04
Text (Strings) Basics
■ W H AT  Y O U  W I L L L E A R N
Any text in Python is called a string — letters, words, even emojis, all wrapped in quotes. You can use
double ""  or single ''  quotes; just be consistent.
You can join strings together with + . This is called concatenation. "Hello " + "there"  becomes 
"Hello there" . Notice the space inside the quotes — Python adds nothing extra for you.
You can also repeat a string with * . "ha" * 3  gives "hahaha" .
■ C O D E  E X A M P L E
first = "Code"
last = "Quest"
title = first + " " + last
print(title)
print("-" * 9)
The line of dashes is made by repeating a string.

Comments & Getting Input from the User
■ W H AT  Y O U  W I L L L E A R N
A comment is a note for humans that Python ignores. Start it with # . Comments explain why your code
does something — they make your program easier to understand later.
To make a program interactive, use input() . It pauses the program, waits for the user to type
something, and hands that text back to you. You usually store it in a variable.
Important: input()  always gives you a string, even if the user types numbers. We will fix that next
week.
■ C O D E  E X A M P L E
# Ask the player for their name
name = input("What is your name? ")
print("Welcome, " + name + "!")
The text inside input() is the question shown to the user.

DAY 07
Changing Data Types (Type Conversion)
■ W H AT  Y O U  W I L L L E A R N
Remember: input()  always gives a string. If the user types 5 , Python sees the text "5", not the
number 5. You cannot do math on text.
To fix this, convert it. int()  turns text into a whole number. float()  turns it into a decimal number. 
str()  turns a number back into text.
This is like translating between two languages so the parts of your program can understand each other.
■ C O D E  E X A M P L E
age_text = input("Your age? ")
age = int(age_text)
next_year = age + 1
print("Next year you will be", next_year)
int() unlocks math; without it, age + 1 would crash.

DAY 08
String Superpowers (Methods & f-strings)
■ W H AT  Y O U  W I L L L E A R N
Strings come with built-in methods — little actions you attach with a dot. .upper()  shouts in capitals,
.lower()  whispers in small letters, .strip()  removes extra spaces, and .title()  Capitalises Each
Word.
The best way to build sentences is an f-string. Put an f  before the quotes, then drop variables straight
inside { } . No more messy +  signs.
■ C O D E  E X A M P L E
name = input("Name? ").strip().title()
print(f"Hello, {name}! Your name has {len(name)} letters.")
f-strings make text clean and easy to read.

DAY 09
More Math (Operators & Order)
■ W H AT  Y O U  W I L L L E A R N
Beyond the basics, Python has three more handy operators. **  means power ( 2 ** 3  = 8). //  is 
floor division — it divides and throws away the remainder. %  (modulo) gives you only the remainder.
Modulo is secretly very useful: number % 2  is 0 for even numbers and 1 for odd ones. You will use it
constantly.
Python follows math order: brackets first, then powers, then multiply/divide, then add/subtract.
■ C O D E  E X A M P L E
total = 17
boxes = total // 5
leftover = total % 5
print(f"{boxes} full boxes, {leftover} left over")
// and % work as a team to split things into groups.

DAY 11
True and False (Booleans)
■ W H AT  Y O U  W I L L L E A R N
A boolean is the simplest possible value: it is either True  or False . Think of it as a light switch — on or
off, yes or no.
Booleans are how the computer answers questions. "Is the player alive?" → True or False. "Is the door
locked?" → True or False.
Note the capital letters: Python wants True  and False , not "true" or "false". Those special words are
the on/off switches of all decision-making.
■ C O D E  E X A M P L E
is_raining = True
has_umbrella = False
print("Raining?", is_raining)
print("Umbrella?", has_umbrella)
Booleans store yes/no facts about your program.

DAY 12
Comparing Things
■ W H AT  Y O U  W I L L L E A R N
Booleans usually come from comparisons. Python can compare values and hand back True or False.
The comparison operators are: ==  equal to, !=  not equal, >  greater than, <  less than, >=  greater-
or-equal, <=  less-or-equal.
Watch out: one =  stores a value; two ==  asks a question. Mixing them up is the most common
beginner mistake.
■ C O D E  E X A M P L E
score = 75
print(score > 50)      # True
print(score == 100)    # False
print(score != 75)     # False
Each comparison produces a single boolean answer.

DAY 13
The if Statement
■ W H AT  Y O U  W I L L L E A R N
The if  statement runs a block of code only when a condition is True. It is the program saying "if this is
the case, then do this."
Structure: write if , then a condition, then a colon : . The lines you want to run go indented underneath
(usually 4 spaces). Indentation is how Python knows what belongs inside the if .
■ C O D E  E X A M P L E
temperature = 35
if temperature > 30:
print("It is hot!")
print("Drink water.")
The two indented lines only run if the condition is True.

DAY 14
if / elif / else
■ W H AT  Y O U  W I L L L E A R N
One condition is rarely enough. elif  (short for "else if") lets you check more conditions in order. else
is the catch-all — it runs when nothing above matched.
Python checks each branch from top to bottom and runs the first one that is True, then skips the rest. So
put your most specific checks first.
■ C O D E  E X A M P L E
score = 82
if score >= 90:
print("Grade A")
elif score >= 75:
print("Grade B")
else:
print("Keep practising")
Only one branch ever runs — the first match wins.

DAY 15
Combining Conditions (and, or, not)
■ W H AT  Y O U  W I L L L E A R N
Sometimes a decision depends on several things at once. The logical operators join conditions together.
and  is True only when both sides are True. or  is True when at least one side is True. not  flips a
boolean to its opposite.
Example: a player can enter a vault if has_key and knows_password  — both are required.
■ C O D E  E X A M P L E
has_key = True
has_torch = False
if has_key and has_torch:
print("Enter the cave")
else:
print("You are missing something")
'and' demands both; change one to False and the result flips.

DAY 16
The while Loop
■ W H AT  Y O U  W I L L L E A R N
A while loop repeats a block of code as long as a condition stays True. It is like saying "keep doing this
while there is still work to do."
The danger: if the condition never becomes False, the loop runs forever (an infinite loop). So inside the
loop you must change something that moves you toward stopping.
■ C O D E  E X A M P L E
count = 1
while count <= 5:
print("Step", count)
    count = count + 1
print("Done!")
count grows each pass; when it passes 5 the loop stops.

DAY 17
The for Loop and range()
■ W H AT  Y O U  W I L L L E A R N
A for loop repeats a known number of times or steps through a collection. It is cleaner than a while loop
when you know how many rounds you need.
range(5)  produces the numbers 0,1,2,3,4. range(1, 6)  gives 1,2,3,4,5. range(0, 10, 2)  counts in
steps of 2.
The loop variable (often i ) holds the current number on each pass.
■ C O D E  E X A M P L E
for i in range(1, 6):
print(f"Counting: {i}")
for x in range(0, 10, 2):
print(x)
range() is the engine that feeds a for loop its numbers.

DAY 18
break and continue
■ W H AT  Y O U  W I L L L E A R N
Inside a loop you have two emergency controls. break  instantly stops the whole loop and jumps out. 
continue skips the rest of the current round and jumps straight to the next one.
Use break  when you have found what you were looking for. Use continue  when you want to ignore
certain cases but keep looping.
■ C O D E  E X A M P L E
for n in range(1, 10):
if n == 6:
break
# stop completely
if n % 2 == 0:
continue
# skip even numbers
print(n)
Prints 1, 3, 5 — evens skipped, loop ends at 6.

Loop Patterns (Counters & Totals)
■ W H AT  Y O U  W I L L L E A R N
Two patterns appear in almost every program. A counter counts how many times something happens:
start at 0, add 1 each time. A total (accumulator) adds up values: start at 0, add the value each round.
The key idea: create the variable before the loop, update it inside the loop, and use it after the loop.
■ C O D E  E X A M P L E
total = 0
for price in [5, 8, 3, 10]:
    total = total + price
print(f"You spent {total} coins")
total starts at 0 and grows with each price.

DAY 21
What is a List?
■ W H AT  Y O U  W I L L L E A R N
A list is one variable that holds many values in a row. You write it with square brackets [ ]  and commas
between items.
A list keeps its items in order, and it can hold any types — numbers, strings, even other lists.

Think of a list as a backpack: one object, many slots.

inventory = ["sword", "shield", "potion"]

print("Items:", len(inventory))
One variable, three items, kept in order.

DAY 22
Reaching Into Lists (Indexing)
■ W H AT  Y O U  W I L L L E A R N
Each item in a list has a position number called an index. Counting starts at 0, not 1. So the first item is
index 0, the second is index 1, and so on.
Grab an item with square brackets: inventory[0] . Negative indexes count from the end: 
inventory[-1]  is the last item.
■ C O D E  E X A M P L E
team = ["Ana", "Ben", "Cara", "Dia"]
print(team[0])    # Ana (first)
print(team[2])    # Cara
print(team[-1])   # Dia (last)
Index 0 = first. -1 = last. A common stumble — practise it!

Changing Lists (Add & Remove)
■ W H AT  Y O U  W I L L L E A R N
Lists can change — they are flexible. .append(x)  adds an item to the end. .insert(i, x)  slots an
item at position i. .remove(x)  deletes the first matching item, and .pop()  removes the last item and
hands it back.
You can also overwrite an item directly: list[0] = "new" .
■ C O D E  E X A M P L E
bag = ["apple", "bread"]
bag.append("cheese")
bag.remove("apple")
bag[0] = "toast"
print(bag)   # ['toast', 'cheese']
Lists grow, shrink, and change on the fly.

DAY 24
Slicing Lists
■ W H AT  Y O U  W I L L L E A R N
Slicing grabs a section of a list at once. The pattern is list[start:stop] . It includes start  but stops 
before stop .
nums[1:4]  takes items 1,2,3. Leaving a side blank means "to the end" or "from the start": nums[:3]  or 
nums[2:] .
■ C O D E  E X A M P L E
nums = [10, 20, 30, 40, 50]
print(nums[1:4])   # [20, 30, 40]
print(nums[:2])    # [10, 20]
print(nums[3:])    # [40, 50]
Slicing copies a chunk without touching the original.

Tuples — Lists That Cannot Change
■ W H AT  Y O U  W I L L L E A R N

print("X is", point[0])
print("Y is", point[1])
# point[0] = 9  -> this would cause an error
Indexing works like a list; changing does not.

Keys are labels; values are the data behind them.

Working With Dictionaries
■ W H AT  Y O U  W I L L L E A R N
Dictionaries are easy to update. Add or change a value by assigning to a key: player["hp"] = 80 . If the

Loop through one with for key in player:  or get pairs with .items() . Check membership with "hp"

for key, value in player.items():

Functions — Reusable Blocks of Code
So far your code runs once, top to bottom. Functions let you name a block of code and reuse it whenever you
like. They are how big programs stay organised and tidy.
DAY 31
What is a Function?
■ W H AT  Y O U  W I L L L E A R N
A function is a named recipe — a block of code you write once and run many times. You define it with 
def , give it a name, add brackets and a colon, then indent the steps.
Defining a function does not run it. To run it you call it by writing its name with brackets: greet() .
Functions stop you repeating yourself and let you read code like a list of clear actions.
■ C O D E  E X A M P L E
def greet():
print("Welcome, hero!")
print("Your quest begins.")
greet()
greet()
Defined once, called twice — no copy-paste needed.

Defaults make some inputs optional and code friendlier.

DAY 35
Scope — Where Variables Live
■ W H AT  Y O U  W I L L L E A R N
Scope is the area where a variable exists. A variable created inside a function is local — it only lives
during that function call and vanishes afterwards.
A variable created outside all functions is global — visible everywhere. Best practice: keep most
variables local and pass data in through parameters and out through return .
■ C O D E  E X A M P L E
score = 0
# global
def earn():
    bonus = 10
# local, dies after the call
return score + bonus
print(earn())
'bonus' cannot be seen outside the function.

DAY 38
Recursion — Functions That Call Themselves
■ W H AT  Y O U  W I L L L E A R N
Recursion is when a function calls itself to solve a smaller version of the same problem. Every recursion
needs a base case — a simple situation where it stops — or it loops forever.
Classic example: factorial. 5! = 5 * 4! , and so on, until 1! = 1  (the base case).
■ C O D E  E X A M P L E
def factorial(n):
if n <= 1:
return 1
# base case
return n * factorial(n - 1)
print(factorial(5))   # 120
The function shrinks the problem until the base case stops it.

Errors and Exceptions
■ W H AT  Y O U  W I L L L E A R N
When something goes wrong, Python raises an exception and the program crashes. You can catch it
with try  / except .
Put risky code in the try  block. If it fails, the matching except  block runs instead of crashing. This is
how real apps stay alive when users do unexpected things.
■ C O D E  E X A M P L E
try:
    n = int(input("A number: "))
print(10 / n)
except ValueError:
print("That was not a number!")
except ZeroDivisionError:
print("Cannot divide by zero!")
try/except catches problems and keeps the program running.

Wizard inherits hp from Hero and adds cast().

Overriding and super()
■ W H AT  Y O U  W I L L L E A R N
A child class can override a parent method — write a method with the same name and the child's version
wins.
If you still want the parent's version plus extra, call super() . super().__init__(...)  runs the parent's
setup, then you add the child-specific parts.
■ C O D E  E X A M P L E
class Hero:
def __init__(self, hp):
self.hp = hp
class Wizard(Hero):
def __init__(self, hp, mana):
        super().__init__(hp)
self.mana = mana
super() reuses the parent setup, then the child extends it.

Encapsulation (Protecting Data)
■ W H AT  Y O U  W I L L L E A R N
Encapsulation means hiding an object's inner data and only allowing changes through safe methods. In
Python, an underscore prefix like self._hp  signals "please do not touch this directly."
You then provide controlled methods — for example a take_damage()  that never lets HP drop below 0.
The object protects its own rules.
■ C O D E  E X A M P L E
class Hero:
def __init__(self):
self._hp = 100
def take_damage(self, d):
self._hp = max(0, self._hp - d)
def status(self):
return self._hp
Damage goes through a method that enforces the rules.
